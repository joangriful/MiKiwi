RESUMEN EJECUTIVO: ESTRUCTURA DEL SISTEMA QUIZ EN MIKIWI
=========================================================

1. RUTAS RELACIONADAS CON QUIZ
==============================

A) Ruta principal (Público):
   - GET /quiz → Renderiza Quiz.jsx
   - Archivo: routes/web.php (línea 366-368)
   - Nombre: configurador.quiz
   - No requiere autenticación

B) Guardar resultado (Protegido):
   - POST /profile/quiz-result → ProfileController@saveQuizResult
   - Archivo: routes/web.php (línea 131)
   - Nombre: profile.quiz.save
   - Requiere usuario autenticado

C) Perfil con recomendaciones (Protegido):
   - GET /perfil → Usa quiz_result_category para mostrar productos
   - Archivo: routes/web.php (línea 133-190)
   - Nombre: perfil.view
   - Lee el resultado del quiz de la tabla users


2. CONTROLADORES QUE MANEJAN QUIZ
==================================

Archivo: app/Http/Controllers/ProfileController.php

Método: saveQuizResult(Request $request) - Líneas 158-181
- Input: { category: "string" }
- Validación: category (required, string, max:255)
- Acción: Actualiza user->quiz_result_category
- Output: JSON { success: bool, message: string }


3. MODELOS DE BD RELACIONADOS CON QUIZ
======================================

Modelo: User (app/Models/User.php)
- Propiedad $fillable incluye: 'quiz_result_category'
- Relaciones:
  * hasMany(UserAddress)
  * hasMany(Order)
  * hasMany(Review)
  * hasMany(ChatSession)

NO EXISTE modelo específico para Quiz (todo está en tabla users)

FLUJO DEL QUIZ
==============

1. Usuario accede a /quiz (público)
   └─ Renderiza componente React: Quiz.jsx

2. Responde 4 preguntas:
   Pregunta 1: Personalidad (Explorador/a, Sensual, Audaz, Equilibrado/a)
   Pregunta 2: Estado de Ánimo (Tormenta, Aguas Termales, Club, Jardín)
   Pregunta 3: Sentidos (Tacto, Vista, Oído, Olfato)
   Pregunta 4: Deseo (Control, Clímax, Conexión, Compartir)
   
   Cada opción suma puntos a categorías de productos

3. Sistema calcula ganador:
   - Suma todas las puntuaciones
   - Encuentra la categoría con MÁS puntos
   - Define resultCategory

4. Si usuario está logueado:
   └─ POST /profile/quiz-result
   └─ Guarda en BD: users.quiz_result_category

5. Muestra resultado con mensaje personalizado
   └─ Botón para ver /perfil (recomendaciones)

6. En /perfil:
   └─ Lee quiz_result_category del usuario
   └─ Busca en $quizCategoryMap
   └─ Obtiene 4 productos aleatorios de esa categoría
   └─ Renderiza productos recomendados

IMPORTANTE: LIMITACIONES ACTUALES
=================================

1. Solo se guarda una columna: quiz_result_category
   - No hay registro de puntuaciones individuales
   - No hay fecha de cuándo se hizo el quiz
   - No hay historial de cambios

2. Sin modelo específico de Quiz
   - No existe tabla separada para Quiz o QuizResult
   - No hay relaciones Eloquent para el quiz

3. Sin detalles de respuestas
   - No se guardan respuestas individuales
   - No se guardan puntuaciones por categoría

4. Mapeo estático hardcodeado
   - El mapeo categoría quiz → categoría producto está en las rutas
   - Poco escalable
