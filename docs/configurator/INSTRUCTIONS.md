# 3D Character Configurator

You requested a 3D configurator for a mannequin character.
We have implemented this using the existing React/Inertia infrastructure of the MiKiwi project.

## Accessing the Configurator
Go to: **http://your-app-url/mannequin**

## Files Created
1. **resources/js/Pages/Configurador/MannequinConfigurator.jsx**: The main page logic.
2. **resources/js/Components/Configurador/MannequinModel.jsx**: The 3D model handler.
3. **public/models/**: Directory where you must place your 3D model.

## IMPORTANT: Adding the 3D Model
Since we cannot generate a high-quality organic 3D mesh from a 2D image via code, you must provide the `.glb` file.

1. Obtain a generic female mannequin 3D model in `.glb` or `.gltf` format.
2. Rename it to `mannequin.glb`.
3. Place it in the `public/models/` folder of this project.
   - Path: `MiKiwi/public/models/mannequin.glb`

## Customization Features
- **Texture Upload**: You can upload an image to wrap around the 3D model.
- **Color Change**: You can change the base color of the model.
- **Morphing**: If your `.glb` file contains Morph Targets (Shape Keys) named "Weight" and "Muscle", the sliders will work. You can edit the code in `MannequinModel.jsx` to match your specific shape key names.

## Why is this folder here?
This folder (`configurator`) was created to fulfill the request "el .html hazlo en: configurador (carpeta)". Since the project is a Laravel + React app, the actual "HTML" is generated by the view engine. The code resides in `resources/js`, but this folder serves as the documentation entry point.
